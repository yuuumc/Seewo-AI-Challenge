## 1. 项目背景与产品愿景

### 1.1 背景

当前K12教育场景中，"作业批改"是教师日常工作中耗时最长的环节。一名高中数学教师带2个班（约100人），单次作业批改耗时**2.5-4小时**，其中解答题批改占58%，每周合计**5-12小时**花在重复性批改劳动上。而在这大量时间投入背后：

- **反馈质量低**：教师因时间限制，对解答题通常只画"√""×"，学生无法获知"错在哪一步、为什么错"
- **反馈严重滞后**：从"学生做完"到"拿到批改结果"间隔**1-3天**，错过最佳纠错窗口
- **学情数据空白**：教师凭记忆记住Top 3-5高频错题，无法系统性地归集错题、分析每个学生的薄弱知识点、做精准的个性化辅导
- **教研管理失据**：教研组长靠Excel手工统计成绩、靠"翻本子"抽查批改质量，数据粒度粗、决策靠经验

与此同时，AI技术在OCR识别、数学公式解析、自然语言生成等领域已取得突破性进展——七天网络×阿里云实现单题0.3秒批改、准确率95%+；深圳龙华区教师借助科大讯飞系统日均节省1.5小时批改时间。**技术可行性已被验证，市场正在从早期采用者向早期大众跨越。**

### 1.2 产品愿景

> **"每个学生都被认真对待。"**

打造希沃智教π——一款面向K12学校的AI智能作业批改系统，覆盖多学科（数学为首发学科），实现从"作业布置→学生作答→AI自动批改→个性化评语→知识点薄弱分析→错题归集→变式题推荐→学情报告"的全链路闭环。让教师从重复性批改劳动中解放，将精力聚焦于教学设计与个性化辅导；让每个学生获得即时、精准、有温度的作业反馈；让教研管理者基于数据而非经验做出教学决策。

### 1.3 产品定位一句话

> 希沃智教π是专为K12教师打造的**AI作业批改助手**——拍照即批改、批改即分析、分析即推送。

### 1.4 核心价值主张

| 角色 | 核心价值 |
|------|---------|
| **一线教师** | 批改时间减少60%+，每道解答题获得步骤级诊断与个性化评语，错题自动归集 |
| **教研组长** | 年级学情数据实时可见，精准定位知识薄弱点，基于数据而非经验驱动教研决策 |
| **学生** | 提交即获反馈（≤5分钟），知道"错在哪、为什么错、怎么改"，错题自动归档推送变式练习 |
| **信息化主任** | 零学习成本嵌入现有教学流程，使用数据自动生成效果报告，降低"买而不用"的风险 |
| **学校管理者** | 全校教学质量数据可视化，教学管理从"经验驱动"升级为"数据驱动" |

## 2. 市场环境与竞品格局

### 2.1 市场规模

- 全国K12学校约**21.6万所**，在校生约**2.9亿人**
- K12教育信息化市场2025年规模约**5000亿元**，年增速约8%
- AI+教育赛道2025年市场规模约**300亿元**，其中AI批改/智慧作业是增长最快的细分领域之一

### 2.2 竞争格局

| 竞品 | 定位 | 优势 | 劣势 |
|------|------|------|------|
| **科大讯飞星火批阅机** | 硬件+软件一体机 | OCR识别强、政府关系深、落地案例多 | 价格高（硬件采购模式）、与希沃硬件生态不兼容 |
| **七天网络×阿里云** | 全科智批改SaaS | 通义千问大模型、批改速度快（0.3秒/题） | 以考试阅卷为主、日常作业场景覆盖弱 |
| **希沃飞象智能作业** | 希沃生态内作业系统 | 与希沃白板/智慧黑板深度集成 | 当前以作文批改见长、数学理科批改能力待补强 |
| **小猿搜题/作业帮** | C端搜题工具 | 题库大、用户基数大 | 以"搜答案"起家、学校端信任度低 |
| **一起教育/云思智学** | 智慧作业平台 | 校本作业资源丰富 | 侧重小学、高中理科深度不够 |

### 2.3 希沃智教π的差异化机会

1. **希沃硬件生态卡位**：全国超**200万间**教室装配希沃设备（智慧黑板/交互智能平板），智教π天然嵌入教师日常教学场景——课件→作业→批改→讲评一气呵成
2. **"教学评"一体化**：竞品多为独立作业工具，希沃可打通"备课（希沃白板）→授课（智慧黑板）→作业（智教π）→学情（数据看板）"全链路
3. **首发数学深度打磨**：数学是K12最痛学科（解答题批改复杂度最高），做透数学=建立最深的竞争壁垒
4. **教研管理视角**：竞品多在"教师工具"层面竞争，希沃可向上延伸到教研组长/教务管理的"数据决策"层，提升客单价和续费黏性

## 4. 核心用户故事

### 4.1 教师 — 日常作业批改（Happy Path）

> **张老师周二下午5:30布置了一份"函数单调性"的课后作业（10道选择题+5道填空题+3道解答题），共102名学生提交。她用希沃智教π扫描/拍照上传学生纸质作业，系统自动识别手写字迹和数学公式，客观题即时完成批改并汇总正误率，解答题按步骤逐行分析给出AI初评结果和个性化评语建议。张老师快速浏览全班的批改进度仪表盘，对几道AI不确定的解答题做了人工确认/修正（约占总题量的15%），一键发布批改结果。学生当晚8点在手机/平板上收到批改反馈和错题订正提醒。整个过程张老师总耗时35分钟（原来需要2.5-3小时），全班学情数据自动归集。**

### 4.2 教研组长 — 月度学情分析

> **李组长打开智教π学情看板，选择了"高一年级·函数导数章节·近30天"。系统展示：全年级562名学生的知识点掌握热力图、"利用导数判断函数单调性"知识点错误率68%（其中"漏讨论a=0情况"占错误原因的43%）、10个平行班的正答率横向对比、5位数学教师的批改行为数据（批注覆盖率、错因标注率、订正追踪率）。李组长据此确定了本周集体备课的主题——"如何突破含参函数单调性讨论"，并定位到3班的王老师（批注覆盖率仅32%）需要针对性指导。从打开系统到形成分析结论，总耗时20分钟（原来需要8-12小时手工统计）。**

### 4.3 学生 — 获得个性化反馈

> **学生小林周三晚自习做完作业后拍照上传，5分钟内收到批改结果。他发现自己3道解答题中第2题做错了——系统在步骤③"求导得f'(x)=3x²-6x+2=0 → 解得x=1±..."处标记了红色，提示"计算错误：此处二次方程求根公式应用有误，判别式应为36-24=12而非36-24=8，正确结果为x=1±(√3/3)"。系统同时推荐了一道同类型的变式练习题和一段3分钟的知识点微课。小林立即订正并完成了推送的变式题。系统自动标记"本题订正通过"。**

### 4.4 信息化主任 — 效果评估与续费论证

> **学期末，王主任登录智教π管理后台，一键导出"2026年秋季学期AI批改使用报告"：全校86%的数学教师持续使用、累积批改作业12.7万份、教师月均节省批改时间18.5小时、学生作业订正率从38%提升至79%、期中考试数学平均分同比提升4.3分。王主任将这份自动生成的报告作为续费论证材料提交校长办公会，预算申请全票通过。**

### 5.2 功能详细说明

#### F1 — 客观题自动批改（P0·MVP必做）

| 项 | 内容 |
|------|------|
| **描述** | 对学生作业中的选择题、填空题、判断题进行全自动批改，判断正误，标注正确答案 |
| **输入** | 学生手写作答的纸质作业照片 / 在线直接作答 |
| **处理** | OCR识别手写答案 → 与标准答案/答案集合匹配 → 判对错 |
| **输出** | 逐题正误标记、正确率统计、错误选项/答案分布 |
| **验收标准** | ①手写数字/字母/符号识别准确率≥98% ②单题批改耗时≤3秒 ③支持同一填空题多空分别判分 ④支持等价答案智能匹配（如"x=2"与"2"、"±√2"与"正负根号2"） |
| **不在此范围** | 数学公式OCR（归入F3）、解答题步骤验证（归入F2） |

#### F2 — 解答题AI辅助批改（P0·MVP必做）

| 项 | 内容 |
|------|------|
| **描述** | 对解答题进行步骤级分析，识别每一步的正确性，定位错误步骤，给出错因分类 |
| **输入** | 学生手写作答的解答题（含文字、公式、图表、证明过程） |
| **处理** | OCR识别全篇→公式解析→题目结构理解→步骤切分→逐步推理验证→错因分类→评语生成→综合评分 |
| **输出** | ①步骤级正误标记（在错误步骤处标红）②错因分类标签（计算错误/概念混淆/逻辑跳跃/步骤遗漏/格式问题）③AI评语建议（自然语言，含改进方向）④综合评分建议（0-满分） |
| **验收标准** | ①解答题AI批改准确率≥90%（以教师复核修正率为准，修正率≤10%）②单道解答题批改耗时≤30秒 ③每一步正确/错误判定准确率≥85% ④错因分类准确率≥80% ⑤所有AI判定结果需标注"AI批改"标识，教师可一键修改 ⑥AI不确定的批改结果（低置信度<0.7）需高亮提醒教师人工复核 |
| **不在此范围** | 完全替代教师批改（AI是辅助而非替代）、几何作图题的图形识别（V1.1） |

#### F3 — 手写数学公式OCR（P0·底层能力）

| 项 | 内容 |
|------|------|
| **描述** | 识别学生手写的数学公式（分式、根号、积分、求和、矩阵、函数符号等），转换为结构化表示 |
| **输入** | 含数学公式的纸质作业照片（支持不同光线、角度、纸张质量） |
| **输出** | LaTeX/结构化数学表达式 + 文本内容 |
| **验收标准** | ①常见高中数学公式（分式/根号/指数/对数/三角/积分/求和/极限/向量/集合）识别准确率≥92% ②复杂公式（矩阵、分段函数、连等推导）识别准确率≥85% ③支持中英文+公式混排 ④对低质量照片（模糊/倾斜/阴影）有预处理能力并给出"照片质量不足"提示 |
| **不在此范围** | 几何图形的形状与位置关系识别（V1.1）、化学方程式/分子式（V2） |

#### F4 — 个性化评语生成（P1）

| 项 | 内容 |
|------|------|
| **描述** | 基于学生作答情况和错因分析，自动生成个性化、有温度的评语 |
| **输入** | 学生本次作业的批改结果+该学生历史的学情数据 |
| **输出** | 自然语言评语（含称赞、问题指出、改进建议、鼓励） |
| **验收标准** | ①评语包含：做得好的地方（正向反馈）+ 需要改进的地方（建设性批评）+ 具体行动建议（可操作）②评语风格可选（鼓励型/严格型/简洁型）③教师可编辑/追加/替换AI评语④评语不出现"AI生成"字样（教师审核后以教师名义发布）⑤无歧视性、打击性、标签化语言 |

## 6. 非功能需求

### 6.1 性能

| 指标 | 目标值 | 测量方式 |
|------|--------|---------|
| 客观题单题批改耗时 | ≤3秒 | 从照片上传完成到批改结果显示 |
| 解答题单题批改耗时 | ≤30秒 | 从照片上传完成到AI初评结果显示 |
| 100份作业批改（含解答题）总耗时 | ≤10分钟 | 从全部上传完成到批改结果就绪 |
| 学情报告生成耗时 | 年级500人×30天数据≤60秒 | Dashboard加载完成 |
| 页面首屏加载时间 | ≤3秒（4G网络） | Web端/移动端 |
| API响应时间（P95） | ≤2秒 | 非批改类API |
| 并发批改支持 | ≥200份作业同时处理 | 单校使用场景 |

### 6.2 可用性

| 指标 | 目标值 |
|------|--------|
| 系统可用性（SLA） | ≥99.5%（月度） |
| 计划内维护窗口 | 每周六凌晨2:00-4:00 |
| 新教师首次完成批改操作的学习时间 | ≤15分钟（含简短视频教程观看） |
| 批改操作步骤数 | 拍照→上传→等待→查看结果→（可选）修正→发布，核心步骤≤3步 |
| 移动端适配 | 教师端支持微信小程序，学生端支持微信小程序+Web |

### 6.3 兼容性

| 维度 | 要求 |
|------|------|
| 浏览器 | Chrome/Firefox/Edge 最近2个大版本 |
| 移动端 | iOS 15+ / Android 10+ |
| 希沃硬件生态 | 与希沃智慧黑板、交互智能平板无缝集成（单点登录+数据互通） |
| 第三方对接 | 支持对接：主流教务管理系统（API对接）、企业微信/钉钉/飞书（消息推送）、希沃白板（课件导入导出） |

### 6.4 安全与合规

| 维度 | 要求 |
|------|------|
| 等保合规 | 通过等保三级测评 |
| 数据加密 | 传输层TLS 1.3，存储层AES-256 |
| 数据主权 | 合同明确：所有教学数据归学校所有，学校可随时全量导出 |
| 数据脱敏 | 学生姓名在学情看板中默认部分脱敏（如"张*三"），支持根据角色控制脱敏级别 |
| 权限控制 | RBAC五级角色体系，最小权限原则 |
| 审计日志 | 所有数据访问操作记录审计日志，保存≥180天，不可删除 |
| 个人信息保护 | 响应《个人信息保护法》：提供数据导出、删除、撤回同意等功能；家长端提供数据知情与授权管理入口 |
| 部署方式 | 支持公有云SaaS（默认）/ 混合云 / 本地化私有部署（大型学校/教育局需求） |

### 6.5 可扩展性

| 维度 | 要求 |
|------|------|
| 多学科扩展 | 学科知识体系设计为可配置的标签框架，新增学科不涉及架构改动 |
| 题库容量 | 支持千万级题目存储与检索，题目标签体系可扩展 |
| 用户规模 | 首期支持100所学校并发，架构支持水平扩展至万校级别 |
| 第三方集成 | 提供标准化RESTful API + Webhook，支持学校自定义对接 |

### 6.6 用户体验原则

| 原则 | 要求 |
|------|------|
| **零学习成本** | 教师操作路径不超过现有工作流：拍照→上传→确认→发布 |
| **AI辅助而非替代** | 所有AI结果必须可被教师修改、驳回，系统永远以教师确认为最终结果 |
| **教师控制节奏** | 教师决定何时批改、何时发布结果、何时推送给学生——系统不越权 |
| **透明可解释** | 每项AI判断标注置信度；低置信度结果主动提醒人工复核 |
| **渐进增强** | 新学校默认开启客观题自动批改，解答题AI批改由教师手动开启，避免"一下全开→不信任→弃用" |

## 8. MVP范围与迭代路线图

### 8.1 MVP定义（V1.0 — "能用且好用"）

**核心假设**：如果数学教师能用智教π在30分钟内完成原来2.5小时的批改工作，且批改质量不下降（甚至更精细），他们就会持续使用。

**MVP范围**：

| 包含（IN） | 不包含（OUT） |
|------------|---------------|
| ✅ 数学学科（高中数学首发） | ❌ 其他学科（英语、语文、物理等） |
| ✅ 客观题全自动批改（选择/填空/判断） | ❌ 几何图形题自动批改 |
| ✅ 解答题AI辅助批改（步骤级分析） | ❌ 全自动无人值守批改 |
| ✅ 手写数学公式OCR | ❌ 化学方程式/分子式OCR |
| ✅ 错题自动归集+班级学情统计 | ❌ 年级学情仪表盘（教研端） |
| ✅ 教师端Web + 学生端微信小程序 | ❌ 教研管理端、管理后台完整版 |
| ✅ 希沃白板基础互通（单点登录） | ❌ 深度集成（课件-作业-讲评联动） |
| ✅ 基础题库（1000+高中数学题，含知识点标签） | ❌ 变式题自动推荐 |
| ✅ 基础用户管理（手动导入/Excel导入） | ❌ 教务系统API自动对接 |
| ✅ SaaS公有云部署 | ❌ 混合云/私有化部署 |

**MVP成功标准**：
- 10所学校、≥50名教师持续使用≥8周
- 教师单次批改耗时减少≥50%
- 周活教师率≥60%
- 教师NPS≥30

## 9. 约束条件与假设

### 9.1 约束条件

| 约束 | 说明 | 影响 |
|------|------|------|
| **必须融入希沃生态** | 需与希沃白板、智慧黑板实现登录互通与数据互通 | 技术架构需兼容希沃现有技术栈与API规范 |
| **公立校采购周期长** | 政府招标流程通常3-6个月 | 种子学校优先选择私立/民办学校，缩短验证周期 |
| **教师技术能力差异大** | 年龄偏大教师对新技术接受度低 | UX设计须极致简单，提供"手把手"式新手引导 |
| **数据合规硬红线** | 等保三级+个人信息保护法合规 | 技术选型和架构设计须从Day 1考虑合规 |
| **手写公式OCR挑战** | 高中生手写质量差异大（整洁↔潦草），数学公式复杂度高 | OCR准确率是批改质量的天花板，需持续投入优化 |
| **网络环境差异** | 部分学校网络基础设施较差（尤其县域高中） | 需支持离线批改场景（缓存+异步上传） |

### 9.2 关键假设

| # | 假设 | 验证方式 | 风险程度 |
|---|------|---------|---------|
| H1 | 教师愿意将批改工作从"全手动"转变为"AI辅助+人工复核" | MVP种子用户深度访谈，观察实际工作流变化 | 🟡 中 |
| H2 | 数学解答题AI批改准确率能达到≥90%（教师修正率≤10%） | 用真实作业数据做离线评测+上线后Tracking修正率 | 🔴 高 |
| H3 | 学生使用手机/平板查看批改结果和提交订正在大多数高中是被允许的 | 种子学校调研确认设备政策 | 🟡 中 |
| H4 | 学校信息化预算中能为"AI批改"单独立项（年费10-20万/校） | 种子学校合同价格验证 | 🟡 中 |
| H5 | 希沃品牌在K12学校的信任度足够支撑新产品快速获客 | 种子学校招募周期和转化率跟踪 | 🟢 低 |
| H6 | 教师节省下来的批改时间会用于更有价值的教学行为（而非被其他非教学事务挤占） | 教师时间日志追踪（使用前后对比） | 🟡 中 |

## 10. 风险与应对

| # | 风险 | 概率 | 影响 | 应对策略 |
|---|------|------|------|---------|
| R1 | **解答题AI批改准确率不达标**——教师不信任系统，修正率>20%，最终弃用 | 中 | 致命 | ①MVP阶段解答题批改定位为"AI初评+人工复核"，不承诺全自动；②低置信度结果强制提醒人工复核，建立信任；③建立批改质量持续反馈机制（教师修正=训练数据），准确率随使用量提升 |
| R2 | **教师"买而不用"**——系统部署后使用率低，学校不续费 | 中 | 致命 | ①UX极简设计，批改操作步骤≤3步；②客观题先跑通→建立信任→再推解答题；③教研组长端提供"使用率看板"，自上而下推动；④种子期提供专属客户成功经理，陪跑使用 |
| R3 | **手写公式OCR识别率低**——导致后续批改全链条失效 | 中 | 高 | ①设定照片质量检测机制，低质量照片拒绝接收并提示重拍；②提供"拍照引导"（拍摄框/示例图）；③OCR+LLM多模型融合，一个模型识别不清时fallback到另一模型；④持续收集真实手写数据做模型微调 |
| R4 | **学校禁止学生使用手机**——学生端无法触达，反馈闭环断裂 | 低 | 高 | ①MVP优先覆盖允许学生使用手机/平板的学校（如寄宿制学校）；②提供"纸质批改结果打印"兜底方案；③与学校协商"仅限作业场景"的设备使用政策 |
| R5 | **数据隐私合规事故**——学生数据泄露引发舆论危机 | 低 | 致命 | ①Day 1即通过等保三级；②建立数据安全专项团队（或外包安全审计）；③数据脱敏+最小权限+审计日志三件套；④制定数据泄露应急响应预案 |
| R6 | **竞品（科大讯飞/七天网络）降价/免费策略打击** | 中 | 高 | ①依托希沃硬件生态建立转换成本——智教π与希沃白板/智慧黑板深度集成，单独竞品无法替代；②向上延伸教研管理场景（竞品弱项），提升决策者黏性 |

### A. 术语表

| 术语 | 定义 |
|------|------|
| 解答题 | 需要写过程/步骤的主观题，如证明题、计算题，区别于只写结果的选择题/填空题 |
