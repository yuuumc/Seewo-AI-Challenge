### 学生端设计原则（来自用户研究）

| 原则 | 说明 |
|------|------|
| **不是ChatGPT** | Math Coach 是苏格拉底式教练——追问、引导、给提示，不直接给答案 |
| **不是传统错题本** | AI重构错误——不只告诉你"哪错了"，而是"为什么错、根因是什么、知识链哪里断了" |
| **不是列表** | 知识树用颜色和树形展示掌握度，一眼看到强弱分布 |
| **不是看分数** | 成长报告看趋势、看变化、看AI预测——从"看成绩"到"看成长" |

## 2. 核心交互流程

### 2.1 教师批改流程（核心Happy Path）

```
上传作业照片
    │
    ▼
OCR识别（手写体+公式）
    │
    ▼
AI批改引擎逐题分析
    ├── 客观题：答案匹配 → 自动判对错 → 汇总正误率
    └── 解答题：步骤切分 → 逐步验证 → 错因分类 → 生成评语
    │
    ▼
批改结果展示（按学生/按题目两种视图）
    │
    ▼
教师复核（重点关注AI低置信度结果）
    ├── 确认 ✓ → 保持不变
    └── 修正 ✏️ → 修改判定/追加批注 → 反馈至模型（训练飞轮）
    │
    ▼
一键发布 → 学生端即时可见
    │
    ▼
学情数据自动归集（错题+知识点+错误类型）
```

### 2.2 学生侧流程

```
收到批改通知 → 查看逐题详情 → 查看步骤级诊断
    │
    ▼
错题自动进入错题本 → 提交订正 → 系统自动验证
    │
    ├── 订正正确 → 标记"已掌握"
    └── 订正仍错 → 提醒再订正 → 3次不通过则通知教师
    │
    ▼
系统推送变式题（同知识点·匹配难度）
```

## 3. 关键界面设计原则

| 原则 | 设计体现 |
|------|---------|
| **零学习成本** | 教师操作路径: 拍照→上传→确认→发布，核心≤3步 |
| **AI辅助而非替代** | 所有AI结果可被教师修改/驳回；低置信度结果主动提醒复核 |
| **教师控制节奏** | 教师决定何时批改、何时发布；系统不越权推送 |
| **透明可解释** | 每项AI判断标注置信度；步骤级分析让教师"看到AI的思考过程" |
| **渐进增强** | 新学校默认仅开客观题自动批改，解答题AI批改由教师逐步开启 |

## 5. 设计决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 教师端形态 | Web优先 | 批改工作在办公室用电脑完成，移动端为辅 |
| 学生端形态 | 微信小程序优先 | 学生覆盖率最高，无需安装 |
| 解答题批改交互 | AI初评+低置信度高亮+教师可一键修正 | 避免"AI全自动→教师不信任→弃用" |
| 学情看板下钻 | 年级→班级→知识点→题目→学生 五层 | 满足教研组长"从全局到个体"的分析路径 |
| 评语归属 | 以教师名义发布，AI为"建议"角色 | 维护师生情感联结，符合教师专业尊严 |
