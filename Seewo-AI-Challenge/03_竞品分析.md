# 03 竞品分析

|------|-------------------|-----------------|-----------------|----------------|-------------------|

## 2. 各竞品详析

### 2.1 科大讯飞·星火批阅机
- **模式**：扫描仪硬件 + AI批改软件一体
- **优势**：讯飞在OCR和语音识别领域技术积累深厚，政府关系强，已在全国多地落地
- **劣势**：①硬件采购模式价格高（单台数万元），学校预算门槛高 ②与希沃硬件生态不兼容 ③以"机器"为中心而非"教师工作流"为中心
- **对我方的启示**：AI批改技术可行性已被验证，但要避免硬件模式

### 2.2 七天网络×阿里云·全科智批改
- **模式**：通义千问大模型驱动的在线阅卷SaaS
- **优势**：批改速度快（0.3秒/题），大模型泛化能力强，考试场景深耕
- **劣势**：①以"考试阅卷"为核心场景，日常作业的拍照上传、手写识别体验弱 ②独立平台，不与教学系统集成
- **对我方的启示**："批改速度"是可量化的竞争力，但场景决定产品形态

### 2.3 希沃飞象智能作业（兄弟产品）
- **模式**：希沃生态内的作业管理工具
- **优势**：与希沃白板/智慧黑板深度融合，入口便捷
- **劣势**：当前以英语作文/语文作文批改见长，数学理科的解答题批改能力弱
- **与本产品关系**：智教π定位为数学理科批改的"专业补位"，与飞象形成学科互补

### 2.4 小猿搜题/作业帮（C端竞品）
- **模式**：拍照搜题→匹配题库答案
- **优势**：海量题库、C端用户基数大、学生使用习惯已养成
- **劣势**：①"搜答案"的基因导致学校端信任度低（教师担心学生抄答案）②没有"批改"概念——只给答案不给诊断 ③不产生教师端的价值
- **对我方的启示**：学校端产品必须让教师"可控、可见、可信"

## 3. 智教π的差异化定位

```
        考试/阅卷
           ↑
  七天网络 ●
           │
  B端 ─────┼─────────→ C端
  (学校)   │  ● 小猿搜题 (学生自用)
           │
  希沃智教π ●  ← 日常作业 + 教学闭环
  科大讯飞 ●
           │
           ↓
        日常作业
```

| 差异化维度 | 智教π的做法 |
|-----------|------------|
| **步骤级批改** | 竞品多为"对/错"二元判断，智教π对解答题逐步骤分析+错因分类 |
| **教研管理端** | 竞品聚焦教师个体，智教π向上延伸到教研组长/教务管理 |
| **教学评一体化** | 竞品为独立工具，智教π对接希沃白板→课件→作业→讲评全链路 |
| **首发数学** | 数学是K12最痛学科（解答题占比高、批改复杂度最高），做透即建壁垒 |

## 4. 结论

市场已有先行者验证了"AI批改"的方向，但现有产品要么偏向考试场景（七天网络），要么偏向硬件模式（科大讯飞），要么缺乏理科深度（希沃飞象）。**日常作业场景 × 数学深度批改 × 教研管理视角**的组合存在明显的市场空白，这正是智教π的机会窗口。
