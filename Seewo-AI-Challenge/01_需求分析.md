## 1. 项目背景与产品愿景

### 1.1 背景

当前K12教育场景中，"作业批改"是教师日常工作中耗时最长的环节之一。

- 一名高中数学教师带2个班（约100人）`[调研·48874教师样本]`，单次作业批改耗时约**2.5-4小时**，其中解答题批改占时最长 `[调研·教师工作时间叙事研究]`
- 七天网络×阿里云2025年发布的"全科智批改系统"实测数据：一份数学试卷主观题人工批改平均85.6秒/题 `[案例·2025 WAIC]`
- 每周数学教师批改解答题约需6.2小时 `[案例·七天网络]`
- 反馈延迟：从学生交作业到拿到批改结果通常间隔**1-3天**，错过最佳纠错窗口 `[调研·多校教师访谈]`
- 教师批改时对解答题通常只画"√""×"或给总分，学生无法获知"错在哪一步、为什么错" `[调研·数学作业批改现状研究]`
- 教研组长靠Excel手工统计成绩、靠"翻本子"抽查批改质量，数据粒度停留在"题号正确率"，无法下沉到"错误类型""学生群体聚类" `[调研]`

AI技术可行性方面——

- 七天网络×阿里云：单题批改0.3秒，准确率95%+ `[案例·2025 WAIC]`
- 科大讯飞星火智能批阅机·深圳龙华区：数学教师日均批改时间减少1.5小时 `[案例·2025]`
- 成都七中东部学校：AI批改准确率高达98% `[案例·2025]`

**结论**：技术可行性已被部分验证，但——目前能查到的公开数据均来自厂商自报或合作学校PR稿件，**缺少独立第三方的严格评测**。AI批改在复杂解答题、手写公式等场景的真实准确率，仍需在真实教学中验证 `[关键不确定性]`。

### 1.2 产品愿景

> **"每个学生都被认真对待。"**

打造希沃智教π——面向K12学校的AI智能作业批改系统。首发数学学科，实现"作业布置→AI批改→个性化评语→知识点薄弱分析→错题归集→学情报告"的全链路。让教师从重复性批改劳动中解放；让每个学生获得即时、精准的作业反馈；让教研管理者基于数据而非经验决策。

### 1.3 产品定位一句话

> 希沃智教π是专为K12教师打造的**AI作业批改助手**——拍照即批改、批改即分析、分析即推送。

### 1.4 核心价值主张

| 角色 | 核心价值 | 数据来源 |
|------|---------|---------|
| **一线教师** | 批改时间目标减少50%+，每道解答题获得步骤级诊断，错题自动归集 | `[目标]` 基于已公开案例的效率提升幅度（50%-95%）取保守值 |
| **教研组长** | 年级学情数据实时可见，精准定位知识薄弱点 | `[调研]` 基于教研组长月均8-12小时手工统计数据的痛点 |

| **信息化主任** | 使用数据自动生成效果报告 | `[调研]` 基于信息化主任预算论证难的痛点 |
| **学校管理者** | 教学管理从"经验驱动"升级为"数据驱动" | `[目标]` 产品长期愿景 |

## 2. 市场环境与竞品格局

### 2.1 市场规模 `[调研·第三方行业报告]`

- 全国K12学校约**21.6万所**，在校生约**2.9亿人**
- K12教育信息化市场2025年规模约**5000亿元**
- AI+教育赛道2025年约**300亿元**，AI批改/智慧作业是增长最快的细分领域之一

### 2.2 竞争格局 `[调研·公开资料整理]`

| 竞品 | 定位 | 优势 | 劣势 |
|------|------|------|------|
| **科大讯飞星火批阅机** | 硬件+软件一体机 | OCR识别强、政府关系深、落地案例多 | 价格高（硬件模式）、与希沃生态不兼容 |
| **七天网络×阿里云** | 全科智批改SaaS | 通义千问、0.3秒/题 | 以考试阅卷为主，日常作业场景弱 |
| **希沃飞象智能作业** | 希沃生态内作业系统 | 与希沃白板深度集成 | 以作文批改见长，数学理科能力待补强 |
| **小猿搜题/作业帮** | C端搜题工具 | 题库大、用户基数大 | 以"搜答案"起家，学校端信任度低 |
| **一起教育/云思智学** | 智慧作业平台 | 校本作业资源丰富 | 侧重小学，高中理科深度不够 |

### 2.3 希沃智教π的差异化机会 `[分析推断]`

1. **硬件生态卡位**：全国超**200万间**教室装配希沃设备 `[调研·希沃公开数据]`
2. **"教学评"一体化**：打通"备课→授课→作业→学情"全链路，竞品多为独立工具
3. **首发数学深度打磨**：数学是K12最痛学科，做透数学=建立最高竞争壁垒
4. **向上延伸教研管理**：竞品多在"教师工具"层，希沃可延伸至"数据决策"层

### 5.2 功能详细说明

#### F1 — 客观题自动批改（P0·MVP必做）

| 项 | 内容 |
|------|------|
| **描述** | 对选择题、填空题、判断题全自动批改，判断正误，标注正确答案 |
| **验收标准** | ①手写数字/字母/符号识别准确率≥98% `[目标]` ②单题批改耗时≤3秒 `[目标]` ③支持同一填空题多空分别判分 ④支持等价答案智能匹配（如"x=2"与"2"）——实际等价判断能力取决于题库建设投入 |
| **技术依据** | 选择题/填空题为有明确标准答案的客观题，OCR+答案匹配技术成熟度高，七天网络等已实现0.3秒/题 `[案例]` |

#### F2 — 解答题AI辅助批改（P0·MVP必做）

| 项 | 内容 |
|------|------|
| **描述** | 对解答题进行步骤级分析，识别每步正确性，定位错误步骤，给出错因分类。**核心定位是"AI初评+人工复核"而非全自动。** |
| **验收标准** | ①AI批改准确率目标≥90%（教师修正率≤10%）`[目标]` ②单道解答题批改耗时≤30秒 `[目标]` ③每一步正误判定准确率≥85% `[目标]` ④错因分类准确率≥80% `[目标]` ⑤所有AI结果标注"AI批改"，教师可一键修改 ⑥低置信度（<0.7）结果高亮提醒人工复核 |
| **关键不确定性** | 以上准确率目标均为**产品假设**。七天网络自报95% `[案例]` 但场景为考试阅卷（书写相对工整、题目相对标准），日常作业场景（字迹潦草、题目多样）的实际准确率需上线后实测。**这是本项目最高风险假设。** |

#### F3 — 手写数学公式OCR（P0·底层能力）

| 项 | 内容 |
|------|------|
| **描述** | 识别手写数学公式（分式、根号、积分、求和、矩阵、函数符号等），转换为结构化表示 |
| **验收标准** | ①常见高中数学公式识别准确率≥92% `[目标]` ②复杂公式（矩阵、分段函数、连等推导）≥85% `[目标]` ③支持中英文+公式混排 ④低质量照片给出"请重拍"提示 |
| **技术现状** | 公开研究中手写数学公式OCR在标准数据集上的识别率约80-90% `[调研]`，但真实课堂作业场景（潦草字迹、非标准纸张、光照不均）的准确率会显著低于实验室数据 `[关键不确定性]` |

#### F4 — 个性化评语生成（P1）

| 项 | 内容 |
|------|------|
| **描述** | 基于学生作答情况和错因分析，自动生成个性化评语（教师可编辑/追加/替换） |
| **验收标准** | ①评语含：正向反馈+建设性批评+具体行动建议 ②风格可选（鼓励型/严格型/简洁型）③教师始终是评语最终署名者 ④无歧视性、打击性、标签化语言 |

#### F5 — 错题自动归集与学情画像（P0·教师端 / P1·学生端）

| 项 | 内容 |
|------|------|

- 10所学校、≥50名教师持续使用≥8周

## 附录

### A. 术语表

| 术语 | 定义 |
|------|------|
| 解答题 | 需写过程/步骤的主观题（证明题、计算题），区别于只写结果的选择/填空题 |
| 步骤级批改 | 对解答题每步推理/计算分别判断正确性，而非只看最终答案 |
| 错因分类 | 将错误归入预定义类型：计算错误、概念混淆、逻辑跳跃、步骤遗漏、格式问题 |
| 变式题 | 同知识点但题干/数字/问法变化的同类题目 |
| 订正闭环 | "批改→查看→提交订正→验证通过"的完整链路 |
| 学情画像 | 基于历史作业数据的学生各知识点掌握度多维度评估 |
| NPS | Net Promoter Score，净推荐值（0-10） |
| Synthetic Persona | 合成用户画像，基于调研数据整合构建的典型用户，非真实个体 |
