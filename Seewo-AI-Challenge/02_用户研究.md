# 02 用户研究

> 基于全国教师调研数据（48874样本）、教师工作时间叙事研究、K12信息化采购趋势报告的合成分析

## 1. 用户分层

```
        学校管理者（决策者）— 关注教学质量、投入产出比
              ↑
        教研组长（影响者）— 关注学情全局、教研效率、教师成长
              ↑
        一线教师（核心使用者）— 关注省时、好用、批改质量
              ↑
        学生（终端受益者）— 关注即时反馈、错题整理、个性化提升
```

## 2. 一线教师：张老师（合成画像）

| 维度 | 描述 |
|------|------|
| 基本信息 | 32岁，高中数学，教龄8年，带2个班共102人 |
| 日均工时 | 10-11小时（上课+备课+批改+班主任） |
| 批改耗时 | 每次作业2.5-4小时，每周5-12小时 |
| 技术熟练度 | 会用希沃白板、微信、Excel，对好用的新工具持开放态度 |

**批改工作流与其耗时：**

| 步骤 | 耗时 | 痛点指数 |
|------|------|----------|
| ① 选题/组卷 | 25-45分钟 | ★★ |
| ② 布置下发 | 2-3分钟 | ★ |
| ③ 收作业清点 | 5-10分钟 | ★★ |
| ④ 逐题批改（核心） | **90-150分钟** | ★★★★★ |
| ⑤ 评等级/写评语 | 10-15分钟 | ★★★ |
| ⑥ 记录典型错题 | 10-15分钟（常被略过） | ★★★★ |
| ⑦ 课堂讲评 | 15-20分钟 | ★★★ |
| ⑧ 督促订正 | 15-30分钟 | ★★★ |

**核心诉求（按优先级）：**
1. "批解答题最累——每道题的步骤都要逐行看，一道题快则60秒，几何证明题可能2-3分钟"
2. "改完发回去都两三天了，学生早忘了当时怎么想的"
3. "每个学生错在哪、为什么错——100个人我根本记不住"
4. "选择题填空题纯属机械劳动，但数量多，消耗了我40%的时间"

**拒绝理由（需产品化解）：**
- "拍照上传比我直接翻本子还慢怎么办？"
- "AI批的我不放心，错了谁负责？"
- "我还要学一个新系统？太麻烦了"

## 3. 教研组长：李组长（合成画像）

| 维度 | 描述 |
|------|------|
| 基本信息 | 42岁，高中数学教研组长，教龄18年，同时带1个高三班 |
| 月均教研管理工时 | 40-55小时（叠加上课），总工时60-105小时 |

**教研管理工作流与耗时：**

| 步骤 | 月均耗时 | 痛点指数 |
|------|----------|----------|
| ① 制定作业规范 | 2-3小时/学期 | ★★ |
| ② 组织集体备课 | 8-10小时 | ★★★ |
| ③ 校本作业研发 | 15-25小时/学期 | ★★★★ |
| ④ 作业质量抽查 | 3-4小时 | ★★★★ |
| ⑤ 学情数据分析（核心） | **8-12小时** | ★★★★★ |
| ⑥ 组织质量分析会 | 4-6小时 | ★★★★ |
| ⑦ 教师培养指导 | 3-5小时 | ★★★ |

**核心诉求：**
1. "我想知道全年级哪个知识点最薄弱——不是靠猜，是靠数据"
2. "5个老师5种批法，有的只打√×，有的批注写满——怎么统一标准？"
3. "质量分析会上大家都说'多练''抓基础'——这种结论没办法落地"

## 4. 信息化主任：王主任（合成画像）

| 维度 | 描述 |
|------|------|
| 基本信息 | 38岁，信息化中心主任，技术背景 |
| 采购周期 | 选型+招标+部署 6-12个月 |

**核心诉求：**
1. "别买回来没人用——过去十年信息化项目'烂尾'的太多了"
2. "不能是系统孤岛——必须能对接现有的教务系统"
3. "一年十几二十万，怎么跟校长论证值不值？"

**决策关卡：**
- 教师能否在15分钟内学会？
- 有没有同区域类似学校的成功案例？
- 厂商3年后还在吗？数据能迁移吗？
- 等保三级过了吗？

## 5. 关键洞察

| 洞察 | 含义 |
|------|------|
| 解答题批改占据教师58%时间，但反馈质量最低 | 这是AI最该切入的点 |
| 批改→反馈延迟1-3天，学生错过纠错窗口 | 即时反馈是核心价值 |
| 学情分析全靠教师记忆和Excel | AI的规模化数据能力是人工无法替代的 |
| 80%的功能80%的人不用是历史常态 | 产品设计必须从教师现有工作流出发，而非要求教师改变习惯 |
